# MOC - Editores y Terminal

Relaciones y conceptos compartidos entre herramientas de esta categoría.

## Conexiones

### VS Code ↔ Terminal y PowerShell
- La terminal integrada de VS Code puede usar PowerShell como shell por defecto
- Scripts de PowerShell se escriben y depuran en VS Code con la extensión oficial
- Perfiles de terminal en VS Code pueden lanzar PowerShell, CMD o WSL

### Obsidian ↔ Warp
- Ambos usan atajos de teclado personalizables y themes
- Obsidian para documentación, Warp para ejecución de comandos
- Se puede enlazar desde Obsidian a archivos del vault usando wikilinks

### Warp ↔ Terminal y PowerShell
- Warp es una terminal moderna, PowerShell es un shell dentro de ella
- Warp ofrece autocompletado inteligente para comandos de PowerShell
- Workflows de Warp pueden automatizar tareas de PowerShell

## Flujo de Trabajo Recomendado

1. **Planificar** en Obsidian (notas, MOCs, tareas)
2. **Desarrollar** en VS Code (edición, debugging, extensiones)
3. **Ejecutar** en Warp/Terminal (comandos, scripts, automatización)
