# Glosario de Términos

Términos y conceptos que aparecen en múltiples herramientas del vault.

## A

**API (Application Programming Interface)**
Conjunto de reglas que permiten la comunicación entre sistemas. Aparece en: Postman, Playwright, Ollama, GitHub Actions.

**Artefacto**
Archivo generado durante un build o pipeline. Aparece en: GitLab CI/CD, GitHub Actions.

## B

**Backend**
Parte del sistema que maneja lógica de negocio y datos. Aparece en: Node.js, Python, Bases de Datos, Nginx.

**Balanceo de Carga**
Distribución de tráfico entre múltiples servidores. Aparece en: Nginx, PM2 (cluster mode), Docker Swarm.

**Branch (Rama)**
Línea de desarrollo independiente en Git. Aparece en: Git, GitLab, GitHub Actions.

**Bundler**
Herramienta que empaqueta archivos JS/CSS para producción. Aparece en: Bun, Biome (no bundle, pero check unificado).

## C

**CI/CD (Continuous Integration / Continuous Deployment)**
Práctica de integrar cambios frecuentes y desplegar automáticamente. Aparece en: GitLab CI/CD, GitHub Actions, Docker, PM2.

**CLI (Command Line Interface)**
Interfaz de línea de comandos. Aparece en: Terminal, pnpm, Bun, Ollama, Biome, PM2, Warp.

**Cluster**
Grupo de procesos que comparten carga de trabajo. Aparece en: PM2, Redis Cluster, Node.js Cluster.

**Cache**
Almacenamiento temporal de datos para acceso rápido. Aparece en: Redis, Nginx, GitHub Actions (cache), Docker (layer cache).

**Contenedor**
Entorno aislado para ejecutar aplicaciones. Aparece en: Docker, VS Code (Dev Containers), PM2 (Docker), Nginx (Docker).

## D

**Dependency Injection**
Patrón donde las dependencias se pasan externamente. Aparece en: Node.js, Python, Biome (config).

**Despliegue (Deploy)**
Proceso de poner una aplicación en producción. Aparece en: PM2, GitLab, GitHub Actions, Docker, Nginx.

## E

**Ecosystem File**
Archivo de configuración para PM2 que define aplicaciones, variables de entorno y opciones de ejecución. Aparece en: PM2.

**Endpoint**
URL específica de una API. Aparece en: Postman, Ollama API, Playwright, Bun.

## F

**Fixture**
Datos o estado predefinido para pruebas. Aparece en: Playwright, Bun (test runner).

**Frontend**
Interfaz de usuario de una aplicación. Aparece en: VS Code, Biome, Bun (bundler), Warp (UI).

## G

**Graceful Shutdown**
Finalización controlada de un proceso. Aparece en: PM2, Docker, Node.js.

**GraphQL**
Lenguaje de consulta para APIs. Aparece en: Postman, Node.js, Python.

## H

**Hook**
Función que se ejecuta en momentos específicos del ciclo de vida. Aparece en: Git (hooks), Playwright (test hooks), pnpm (pnpmfile.js hooks).

## I

**Infrastructure as Code (IaC)**
Gestión de infraestructura mediante archivos de configuración. Aparece en: Docker, Nginx, GitLab CI/CD, GitHub Actions.

## J

**Job**
Unidad de trabajo en CI/CD. Aparece en: GitLab CI/CD, GitHub Actions.

**JSON**
Formato de intercambio de datos. Aparece en: Postman, Ollama, Biome, Node.js.

## L

**Lint / Linting**
Análisis estático de código para encontrar errores. Aparece en: Biome, VS Code (extensiones).

**Load Balancer**
Distribuidor de tráfico. Aparece en: Nginx, PM2.

**Locator**
Estrategia para identificar elementos en una página web. Aparece en: Playwright.

**Lockfile**
Archivo que fija versiones exactas de dependencias. Aparece en: pnpm, Bun, npm.

## M

**Middleware**
Función que se ejecuta en medio del ciclo request-response. Aparece en: Node.js (Express), Bun, Nginx.

**Mock**
Simulación de un objeto o servicio real para pruebas. Aparece en: Playwright, Bun, Postman.

**Monorepo**
Repositorio único que contiene múltiples proyectos. Aparece en: pnpm (workspaces), Git, GitLab.

## P

**Package Manager**
Herramienta que gestiona dependencias de un proyecto. Aparece en: pnpm, Bun, npm.

**Pipeline**
Secuencia automatizada de pasos en CI/CD. Aparece en: GitLab CI/CD, GitHub Actions.

**Plugin / Extensión**
Módulo que añade funcionalidad a una herramienta base. Aparece en: VS Code, Obsidian, Biome, Ollama, Warp.

**Process Manager**
Gestor que mantiene aplicaciones en ejecución. Aparece en: PM2.

**Proxy Inverso**
Servidor que recibe requests y las redirige al backend correspondiente. Aparece en: Nginx.

**Pub/Sub (Publish/Subscribe)**
Patrón de mensajería donde los publicadores envían mensajes sin conocer a los suscriptores. Aparece en: Redis.

## R

**Rate Limiting**
Limitación de requests para prevenir abuso. Aparece en: Nginx, GitLab, GitHub Actions.

**Replica / Replication**
Copia de datos para redundancia. Aparece en: Bases de Datos, Redis.

**Runner**
Agente que ejecuta jobs de CI/CD. Aparece en: GitLab CI/CD, GitHub Actions.

## S

**Self-Hosted**
Servicio ejecutado en infraestructura propia. Aparece en: GitLab, GitHub Actions, Ollama.

**Server Block**
Configuración de un sitio virtual en Nginx. Aparece en: Nginx.

**Snapshot**
Captura del estado de un sistema para comparación futura. Aparece en: Playwright, Bun.

**Store**
Almacén global de paquetes en pnpm. Aparece en: pnpm.

## T

**Test Runner**
Framework para ejecutar pruebas. Aparece en: Playwright, Bun.

**Tracing / Traza**
Registro detallado de la ejecución para depuración. Aparece en: Playwright, PM2.

## V

**Variable de Entorno**
Valor configurable fuera del código. Aparece en: Postman, PM2, Docker, GitHub Actions, GitLab CI/CD.

## W

**Webhook**
Llamada HTTP automática ante un evento. Aparece en: GitLab, GitHub Actions, Postman.

**WebSocket**
Protocolo de comunicación bidireccional. Aparece en: Bun, Nginx, Node.js.

**Workflow**
Conjunto de pasos automatizados en GitHub Actions. Aparece en: GitHub Actions.

**Workspace**
Entorno que agrupa múltiples proyectos. Aparece en: pnpm, VS Code, Warp.

## Z

**Zero-Downtime Deploy**
Despliegue sin interrupción del servicio. Aparece en: PM2, Nginx, Docker.
